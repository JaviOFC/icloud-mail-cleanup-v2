<objective>
Build the classification engine that scores every email and assigns it to a tier, plus the checkpoint system for persisting results.

Purpose: This is the core intelligence of Phase 1. It takes the scanner output (messages) and contact profiles (from Plan 02), computes weighted composite scores using 8 signals, enforces protection rules, and assigns every email to Trash/Keep-Active/Keep-Historical/Review. The checkpoint module persists results as JSONL for consumption by Phase 2 (content analysis on Review emails) and Phase 3 (interactive review + execution).

Output: classifier.py with scoring engine and tier assignment, checkpoint.py for JSONL persistence, full test coverage.
</objective>

  <action>
    Create `src/icloud_cleanup/classifier.py` with:

    1. Signal weight constants (matching research spec):
       - CONTACT_WEIGHT = 0.30
       - FREQUENCY_WEIGHT = 0.15
       - RECENCY_WEIGHT = 0.15
       - READ_RATE_WEIGHT = 0.15
       - REPLY_RATE_WEIGHT = 0.10
       - APPLE_CATEGORY_WEIGHT = 0.05
       - AUTOMATION_WEIGHT = 0.05
       - FLAGGED_WEIGHT = 0.05
       - TRASH_THRESHOLD = 0.95
       - KEEP_THRESHOLD = 0.70
       - ACTIVE_RECENCY_DAYS = 180

    2. `compute_signals(message: Message, profile: ContactProfile) -> list[SignalResult]`:
       Computes all 8 signals as SignalResult objects:
       - contact_score: 1.0 if bidirectional, 0.5 if sent-to-only, 0.0 if unknown (weight 0.30)
       - frequency_score: Normalize sender frequency by engagement. High volume + low read rate = low score (newsletter pattern). Use `profile.read_rate * min(1.0, profile.times_received_from / 20)` capped at 1.0 (weight 0.15)
       - recency_score: `math.exp(-0.003 * age_days)` where age_days = (now - message.date_received) / 86400. Lambda ~0.003 gives ~1-year half-life (weight 0.15)
       - read_rate_signal: profile.read_rate directly (weight 0.15)
       - reply_rate_signal: profile.reply_rate directly (weight 0.10)
       - apple_category_signal: Map model_category to value — 0(Primary)=0.8, 1(Transactions)=0.7, 2(Updates)=0.3, 3(Promotions)=0.1, None=0.5 (weight 0.05)
       - automation_signal: 1.0 if automated_conversation==0, 0.0 if >0. Combine with unsubscribe_type: if not None, reduce by 0.5 (weight 0.05)
       - flagged_signal: 1.0 if profile.flagged_count > 0, 0.0 otherwise (weight 0.05)

    3. `compute_confidence(signals: list[SignalResult]) -> tuple[float, str]`:
       Weighted average of all signals. Returns (score, explanation_string).
       Explanation format: "contact=0.90; read_rate=0.75; recency=0.60; ..."

    4. `assign_tier(confidence: float, protected: bool, overridden: bool, profile: ContactProfile, message: Message) -> Tier`:
       Per CONTEXT.md locked decisions:
       - If protected and NOT overridden: CANNOT be Trash
         - If high engagement (confidence > 0.6) AND recent (within ACTIVE_RECENCY_DAYS) AND (read_rate > 0.5 OR reply_rate > 0.1): Keep-Active
         - Else if confidence > 0.4: Keep-Historical
         - Else: Review
       - If NOT protected OR overridden:
         - If (1 - confidence) >= TRASH_THRESHOLD (i.e., trash_confidence = 1 - keep_confidence >= 0.95, meaning keep_confidence <= 0.05): Trash
         - If confidence >= KEEP_THRESHOLD:
           - Apply hybrid Active/Historical split: recent (within 180 days) AND (read_rate > 0.5 OR reply_rate > 0.1) -> Keep-Active, else Keep-Historical
         - Else: Review

       IMPORTANT: The confidence score from compute_confidence represents "keep-worthiness" (higher = more worth keeping). Trash threshold means the INVERSE (1 - confidence) must be >= 0.95, i.e., confidence <= 0.05 for Trash. This ensures the 0.95+ bar for trashing per user decision.

    5. `classify_messages(messages: list[Message], profiles: dict[str, ContactProfile], replied_conv_ids: set[int]) -> list[Classification]`:
       For each message:
       - Look up ContactProfile by LOWER(sender_address)
       - If no profile exists, create a default empty profile (unknown sender)
       - Compute signals via compute_signals()
       - Compute confidence via compute_confidence()
       - Check protection via is_protected() from contacts module
       - Check override via check_protection_override() if protected
       - Assign tier via assign_tier()
       - Build Classification object with current timestamp
       Return list of Classification objects.

    - Test the protection + override + tier assignment matrix exhaustively
    - Test edge cases: unknown sender (no profile), zero-age message (recency=1.0), very old message
