  <name>Task 1: Display module and CLI wiring</name>
  <files>src/icloud_cleanup/display.py, src/icloud_cleanup/cli.py, src/icloud_cleanup/__main__.py</files>
  <action>
    Create `src/icloud_cleanup/display.py` with rich-based display functions:

    1. `scan_with_progress(messages: list[Message], callback: Callable[[Message], T]) -> list[T]`:
       - Wraps iteration with rich Progress bar
       - Uses SpinnerColumn, description, MofNCompleteColumn, TimeRemainingColumn
       - Description: "Scanning..." or accepts custom label
       - Returns list of callback results

    2. `classify_with_progress(messages: list[Message], classify_fn: Callable[[Message], Classification]) -> list[Classification]`:
       - Same progress bar pattern, description "Classifying..."
       - Returns list of Classification objects

    3. `display_scan_stats(stats: dict[str, dict], console: Console | None = None) -> None`:
       - Rich Table with columns: Sender | Count | Size | First Seen | Last Seen
       - Sort by count descending, show top 25 senders
       - Format size as human-readable (KB/MB)
       - Format dates as YYYY-MM-DD
       - Show total row at bottom

    4. `display_tier_summary(classifications: list[Classification], console: Console | None = None) -> None`:
       - Rich Table: Tier | Count | % | Avg Confidence
       - Color-coded tier names: Trash=red, Keep-Active=green, Keep-Historical=blue, Review=yellow
       - Show total row

    5. `display_top_senders(classifications: list[Classification], messages: list[Message], console: Console | None = None) -> None`:
       - For each tier, show a Rich Table with Top 10 senders: Sender | Count | Avg Confidence
       - Group classifications by tier, then by sender_address (requires message lookup)
       - Panel wrapper per tier for visual separation

    Create `src/icloud_cleanup/cli.py` with argparse subcommands:

    1. `create_parser() -> argparse.ArgumentParser`:
       - Main parser with description "iCloud Mail Cleanup -- Metadata Classification"
       - Subcommands: scan, classify, report
       - Global options: `--db PATH` (override Envelope Index path), `--checkpoint PATH` (checkpoint file path, default: `~/.icloud-cleanup/checkpoint.jsonl`), `--verbose` / `-v`

    2. `cmd_scan(args) -> None`:
       - Open DB via open_db(args.db)
       - Get sender stats via get_sender_stats()
       - Display stats table via display_scan_stats()
       - Print total email count and unique sender count

    3. `cmd_classify(args) -> None`:
       - Open DB, scan all messages with progress bar
       - Build contact profiles (get_sent_recipients, get_replied_conversation_ids, build_contact_profiles)
       - Check for existing checkpoint -- if exists and not --full, load and merge (incremental)
       - Classify messages with progress bar
       - Merge with existing checkpoint if incremental
       - Save checkpoint via save_checkpoint()
       - Display tier summary via display_tier_summary()
       - Print checkpoint path

       Classify subcommand extra options:
       - `--full`: Force full reclassification (ignore existing checkpoint)
       - `--debug-scores SENDER`: Dump per-signal breakdown for a specific sender (for weight tuning)

    4. `cmd_report(args) -> None`:
       - Load checkpoint via load_checkpoint()
       - If empty, print error and exit
       - Reload messages from DB (needed for sender lookup in top senders display)
       - Display tier summary
       - Display top senders per tier

    5. `main() -> None`:
       - Parse args, dispatch to cmd_scan/cmd_classify/cmd_report
       - Ensure checkpoint directory exists (create if needed)
       - Handle KeyboardInterrupt gracefully

    Create `src/icloud_cleanup/__main__.py`:
    ```python
    from icloud_cleanup.cli import main
    main()
    ```

    Important implementation notes:
    - Progress bars use `rich.progress.Progress` with the exact column config from research
    - All long-running loops (scanning, classifying) must use progress wrappers per SCAN-03
    - The classify subcommand implements the two-pass strategy (CLAS-03): this IS the metadata-only first pass, with Review tier collecting ambiguous emails for Phase 2
    - Use `pathlib.Path` everywhere per CLAUDE.md preference
    - `--debug-scores` is critical for weight tuning -- dump all 8 signals + final confidence for every message from the specified sender
