  <name>Task 1: MLX batch embedding generator with model fallback</name>
  <files>src/icloud_cleanup/embedder.py, tests/test_embedder.py</files>
  <behavior>
    - Test load_embedding_model: returns (model, tokenizer, model_name) tuple
    - Test load_embedding_model: falls back to MiniLM when primary model fails (mock the import error)
    - Test batch_embed: returns numpy array of shape (N, dim) for N input texts
    - Test batch_embed: applies "search_document: " prefix only for modernbert models
    - Test batch_embed: handles empty string inputs without crashing
    - Test batch_embed: batches correctly when len(texts) is not divisible by batch_size
    - Test batch_embed: output embeddings are normalized (L2 norm close to 1.0)
  </behavior>
  <action>
    Create `src/icloud_cleanup/embedder.py` with two public functions:

    1. `load_embedding_model() -> tuple[Any, Any, str]`:
       - Try loading `mlx-community/nomicai-modernbert-embed-base-4bit` via `mlx_embeddings.utils.load`
       - On any exception, log warning and fall back to `mlx-community/all-MiniLM-L6-v2-4bit`
       - Return `(model, tokenizer, model_name)` -- model_name needed to decide prefix

    2. `batch_embed(texts: list[str], model, tokenizer, model_name: str, batch_size: int = 64, max_length: int = 512) -> np.ndarray`:
       - Apply `"search_document: "` prefix only if `"modernbert"` is in model_name (case insensitive)
       - Process in batches of `batch_size`
       - For each batch: `tokenizer.batch_encode_plus(batch, return_tensors="mlx", padding=True, truncation=True, max_length=max_length)`
       - Run `model(input_ids, attention_mask=attention_mask)` to get outputs
       - Call `mx.eval(outputs.text_embeds)` to force GPU compute before numpy conversion
       - Stack all batch results with `np.vstack`
       - Return `(N, dim)` numpy array

    Module constants:
    - `PRIMARY_MODEL = "mlx-community/nomicai-modernbert-embed-base-4bit"`
    - `FALLBACK_MODEL = "mlx-community/all-MiniLM-L6-v2-4bit"`
    - `DOC_PREFIX = "search_document: "`

    Testing strategy: Since MLX GPU requires hardware, tests should mock `mlx_embeddings.utils.load` and the model/tokenizer. Create a `MockModel` and `MockTokenizer` that return properly shaped numpy arrays simulating model output. Test the batching logic, prefix application, and fallback behavior without requiring actual GPU.

    IMPORTANT: The `mx.eval()` call is MLX's GPU synchronization barrier (forces lazy computation to execute). This is NOT Python's built-in eval() -- it's `mlx.core.eval()`.
  </action>
  <verify>
    <automated>uv run pytest tests/test_embedder.py -x -v</automated>
  </verify>
  <done>embedder.py loads ModernBERT with MiniLM fallback, generates (N, dim) embeddings in GPU batches. Tests verify batching logic, prefix application, fallback, and output shapes using mocks.</done>
</task>

  <files>src/icloud_cleanup/clusterer.py, tests/test_clusterer.py</files>
  <behavior>
    - Test cluster_embeddings: returns numpy array of labels same length as input
    - Test cluster_embeddings: labels contain -1 (noise) and positive cluster IDs
    - Test cluster_embeddings: handles edge case of very few points (less than min_cluster_size)
    - Test label_clusters: returns dict mapping cluster_id to list of top TF-IDF terms
    - Test label_clusters: skips noise cluster (-1)
    - Test label_clusters: handles single-cluster edge case
    - Test derive_content_scores: trash-dominated clusters get low scores
    - Test derive_content_scores: keep-dominated clusters get high scores
    - Test derive_content_scores: mixed/noise clusters get mid-range scores (~0.5)
    - Test derive_content_scores: returns dict mapping index to float in [0, 1]
  </behavior>
  <action>
    Create `src/icloud_cleanup/clusterer.py` with three public functions:

    1. `cluster_embeddings(embeddings: np.ndarray, min_cluster_size: int = 100, min_samples: int = 20) -> np.ndarray`:
       - Use `sklearn.cluster.HDBSCAN(min_cluster_size=min_cluster_size, min_samples=min_samples, metric="cosine", cluster_selection_method="eom")`
       - `fit_predict(embeddings)` returns labels array
       - Log cluster count and noise fraction
       - If noise > 50%, log warning (parameter tuning needed per research pitfall 5)
       - Return labels array (int, -1 = noise)

    2. `label_clusters(texts: list[str], labels: np.ndarray, top_n: int = 5) -> dict[int, list[str]]`:
       - For each non-noise cluster ID: extract member texts
       - Fit `TfidfVectorizer(stop_words="english", max_features=1000, max_df=0.9)` on cluster texts
       - Extract top `top_n` terms by mean TF-IDF score
       - Return `{cluster_id: ["term1", "term2", ...]}`

    3. `derive_content_scores(labels: np.ndarray, existing_tiers: list[Tier]) -> dict[int, float]`:
       - For each cluster: count how many members are Trash vs Keep (Active+Historical) vs Review from Phase 1 tiers
       - Cluster dominated by Keep (>60%): content_score = 0.8
       - Cluster dominated by Trash (>60%): content_score = 0.2
       - Mixed cluster or small cluster: content_score = 0.5
       - Noise points (-1): content_score = 0.5 (neutral -- do not shift their tier)
       - Return `{point_index: score}` for all points
       - This leverages the insight from research: "if a cluster is dominated by Phase 1 Trash-tier emails, assign low content scores"

    Testing: Use synthetic numpy arrays for embeddings (random but reproducible with fixed seed). For HDBSCAN tests, create well-separated clusters in low dimensions so HDBSCAN finds them deterministically. For labeling tests, use known text sets with distinct vocabularies. For content scores, use controlled tier distributions.
  </action>
