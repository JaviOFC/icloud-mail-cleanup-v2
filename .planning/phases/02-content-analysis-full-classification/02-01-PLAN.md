  <behavior>

    - Test parse_emlx_body: extracts text/plain from multipart .emlx (byte-count + RFC822)
    - Test parse_emlx_body: strips HTML for HTML-only emails, removes script/style tags
    - Test parse_emlx_body: truncates to max_chars (default 4000)
    - Test parse_emlx_body: returns None for corrupt/empty/binary files
    - Test parse_emlx_body: handles various charsets (utf-8, latin-1, windows-1252) with errors='replace'
    - Test strip_html: removes tags, collapses whitespace, skips script/style content
    - Test strip_html: handles malformed HTML without crashing (fallback to regex strip)
  </behavior>
  <action>
    Create `src/icloud_cleanup/emlx_parser.py` with three public functions:

    1. `build_emlx_lookup(mail_dir: Path, account_uuid: str) -> dict[int, Path]`:
       - Walk `mail_dir / account_uuid` recursively for `*.emlx` files
       - Skip files where stem contains ".partial" (headers-only, no body)
       - Parse stem as int for ROWID — skip non-numeric stems
       - Return `{ROWID: Path}` dict
       - Default mail_dir = `Path.home() / "Library/Mail/V10"`

    2. `strip_html(html_text: str) -> str`:
       - Use stdlib `html.parser.HTMLParser` subclass (no external library per user decision)
       - Skip content inside `<script>` and `<style>` tags
       - Collapse whitespace with `re.sub(r'\s+', ' ', text).strip()`
       - Fallback: if HTMLParser raises, use crude `re.sub(r'<[^>]+>', ' ', html_text)`

    3. `parse_emlx_body(path: Path, max_chars: int = 4000) -> str | None`:
       - Read first line as byte count, read that many bytes as RFC822 message
       - Use `email.message_from_bytes()` to parse
       - For multipart: walk parts, prefer text/plain, fallback to text/html (strip tags)
       - For non-multipart: detect content type, strip HTML if needed
       - Always decode with `part.get_content_charset() or 'utf-8'` and `errors='replace'`
       - Truncate to max_chars before returning
       - Return None on any exception (log warning, never block)

    Create comprehensive test suite in `tests/test_emlx_parser.py`:
    - Use `tmp_path` fixture to create mock .emlx files with known content
    - Test classes: TestLookupTable, TestHtmlStripping, TestBodyExtraction, TestErrorHandling
    - Mock .emlx format: write byte count line + email.message bytes + empty plist

    Follow research patterns exactly from 02-RESEARCH.md Pattern 1 and Pattern 2.
