<task type="auto" tdd="true">
  <name>Task 1: Review session manager and propagation engine</name>
  <files>src/icloud_cleanup/review.py, src/icloud_cleanup/propagation.py, tests/test_review.py, tests/test_propagation.py</files>
  <behavior>
    - ReviewSession dataclass stores session_id, started_at, decisions (cluster_id -> action), individual_decisions (message_id -> action), propagation_applied, completed flag
    - save_session writes ReviewSession to JSON with atomic write (tmp + os.replace)
    - load_session reads JSON back to ReviewSession, returns None if file doesn't exist
    - ReviewSession tracks which clusters have been reviewed for resume support
    - run_review presents clusters via Rich + questionary: display cluster panel, prompt for action, save after each decision
    - Trash auto-approve: items above 0.98 confidence auto-approved, 0.95-0.98 shown for review
    - Actions: approve, skip, reclassify, split, inspect (per CONTEXT.md locked decisions)
    - find_propagation_targets identifies similar senders after a decision: same domain, common prefix before +/. in local part (aliases), sender display name overlap
    - PropagationSuggestion contains source_sender, target_senders, target_message_ids, reason
  </behavior>
  <action>
    Create `src/icloud_cleanup/review.py`:

    1. `ReviewSession` dataclass:
       - session_id: str (e.g., "review_20260305_143022")
       - version: int = 1
       - started_at: int (unix timestamp)
       - last_updated: int
       - auto_triage_summary: dict | None (from AutoTriageResult)
       - decisions: dict[str, dict] (cluster key -> {"action": str, "timestamp": int})
       - individual_decisions: dict[str, dict] (message_id str -> {"action": str, "timestamp": int})
       - propagation_applied: list[dict]
       - completed: bool = False

    2. `save_session(session: ReviewSession, path: Path) -> None`:
       - Serialize to JSON with atomic write (tmp + os.replace)
       - Update last_updated timestamp before write

    3. `load_session(path: Path) -> ReviewSession | None`:
       - Return None if file doesn't exist
       - Parse JSON back to ReviewSession

    4. `get_session_path() -> Path`:
       - Default: `~/.icloud-cleanup/review_session.json`

    5. `run_review(remaining_classifications, messages, session, console) -> ReviewSession`:
       - Group remaining by cluster_label (None -> "Unclustered")
       - Sort clusters by size (largest first) for maximum impact
       - For each unreviewed cluster:
         a. Display Rich Panel with: cluster label, email count, confidence range, top 3 senders, 5 example subjects
         b. Check trash auto-approve: if all items are Trash tier with confidence > 0.98, auto-approve and print notice
         c. For 0.95-0.98 borderline trash clusters, show for review with a note about borderline confidence
         d. Use `questionary.select()` for action choice: Approve / Skip / Reclassify / Split / Inspect
         e. Handle each action:
            - Approve: mark all cluster emails for deletion
            - Skip: leave as-is
            - Reclassify: prompt for target tier via questionary.select
            - Split: use questionary.checkbox to select individual emails to approve
            - Inspect: paginate individual emails with per-email approve/skip
         f. Save session after each cluster decision
         g. After each decision, call propagation and present suggestions
       - Flush Rich console before each questionary prompt (pitfall from research)
       - Return updated session

    Create `src/icloud_cleanup/propagation.py`:

    1. `PropagationSuggestion` dataclass:
       - source_sender: str
       - target_senders: list[str]
       - target_message_ids: list[int]
       - reason: str (e.g., "Same domain: @example.com")
       - suggested_action: str (same as source decision)

    2. `find_propagation_targets(decided_sender: str, action: str, all_classifications: list[Classification], sender_lookup: dict[int, str], already_decided: set[int]) -> list[PropagationSuggestion]`:
       - Strategy 1 - Domain match: find undecided emails from same @domain (skip common domains like gmail, yahoo, outlook, icloud)
       - Strategy 2 - Alias detection: find senders sharing local-part prefix before +, ., or - (e.g., shop@store.com and noreply@store.com)
       - Strategy 3 - Subdomain match: find senders from subdomains of same base domain (e.g., news.store.com and shop.store.com)
       - Exclude already-decided message_ids
       - Return PropagationSuggestion for each group with > 0 targets

    Tests for review.py: test session save/load roundtrip, test session resume skips already-decided clusters, test trash auto-approve threshold logic. Mock questionary for action testing.
    Tests for propagation.py: test domain match, alias detection, subdomain match, common-domain exclusion.
  </action>
  <verify>
    <automated>cd /path/to/project && uv run pytest tests/test_review.py tests/test_propagation.py -x -q</automated>
  </verify>
  <done>
    - ReviewSession serializes/deserializes correctly to/from JSON
    - Session resume correctly skips already-decided clusters
    - Trash auto-approve threshold (0.98) correctly identifies auto-approvable clusters
    - Propagation finds domain matches, alias matches, subdomain matches
    - Common email domains (gmail, yahoo, etc.) excluded from domain propagation
    - Atomic session file writes prevent corruption on Ctrl-C
  </done>

       - Load checkpoint, load messages from DB
       - Call `generate_report(classifications, messages, output_dir, format)`

    2. **New `review` subcommand** (`cmd_review`):
       - Add to subparsers: `review` with description "Interactive review of classified emails"
       - Flags: `--session PATH` (custom session file), `--resume` (resume existing session), `--reset` (start fresh, discard existing session)
       - Flow:
         a. Load checkpoint + messages from DB
         b. Load or create ReviewSession
         c. Build sender_lookup dict from messages (message_id -> sender_address)
         d. Run auto_triage on Review-tier items, display transparency summary via Rich
         e. Let user inspect auto-resolved items if desired (questionary confirm)
         f. Run run_review on remaining items
         g. After review completes, display summary: N approved for deletion, N skipped, N reclassified
         h. Suggest Claude API fallback for any remaining ambiguous items: show estimate_api_cost, ask for confirmation via questionary.confirm
         i. If confirmed, run classify_ambiguous_batch, show mini-review of results
         j. Save updated checkpoint with all decisions

    3. **New `execute` subcommand** (`cmd_execute`):
       - Add to subparsers: `execute` with description "Execute approved deletions"
       - Flags: `--execute` (required to actually delete, otherwise dry-run), `--batch-size N` (default 100), `--batch-pause N` (default 2.0 seconds), `--restore` (restore previously deleted), `--action-log PATH` (custom log location, default `~/.icloud-cleanup/action_log.db`)
       - Flow:
         a. Load checkpoint + review session
         b. Collect all approved message_ids from session (decisions with action="approve" + propagation-applied)
         c. Load messages from DB for approved items
         d. Create ActionLog
         e. If `--restore`: call restore_from_log
         f. Else: call execute_deletions with dry_run=(not args.execute)
         g. Display Rich summary: N moved to trash, N errors, N skipped (protected)
         h. If dry-run: print "Run with --execute to carry out deletions"

    Register new subcommands in `create_parser()`. Follow existing pattern: each subcommand maps to a `cmd_*` function.
