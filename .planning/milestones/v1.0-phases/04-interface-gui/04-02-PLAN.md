<objective>
Build the Review screen with a two-column split layout: scrollable cluster list on the left, cluster detail panel on the right. Includes multi-select bulk actions, auto-triage trigger, propagation (both inline toasts AND a dedicated Propagation tab), Claude API fallback for remaining ambiguous emails, and session persistence.

Purpose: This is the primary user interaction surface -- where the user reviews classification results, makes approve/skip decisions, reviews propagation suggestions in bulk, and optionally triggers API analysis on remaining ambiguous emails.
Output: Fully functional Review screen with Clusters + Propagation tabs, API fallback action, that loads clusters from checkpoint, supports navigation, selection, approval, and saves decisions to the same session format the CLI uses.
</objective>

       - `on_button_pressed(event)` handler for #btn-api:
         1. Compute remaining ambiguous: Review-tier classifications whose message_id is NOT in any session decision or individual_decision (same logic as cli.py lines 710-726).
         2. If none remaining, notify "No ambiguous emails remaining" and return.

            c. On success: `self.call_from_thread(self.notify, f"Batch submitted: {batch.id}. Processing is async -- re-run TUI later to see results.", severity="information", timeout=10)`.
            d. On error: `self.call_from_thread(self.notify, f"API error: {e}", severity="error")`.
