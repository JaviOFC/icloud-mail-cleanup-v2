  <action>
    Create `src/icloud_cleanup/contacts.py` with:

    1. `build_contact_profiles(messages: list[Message], sent_recipients: dict[str, dict], replied_conv_ids: set[int]) -> dict[str, ContactProfile]`:
       - Group messages by LOWER(sender_address)
       - For each sender, compute:
         - times_received_from: count of messages
         - last_received_from: max date_received
         - read_rate: sum(read) / count
         - reply_rate: count messages where (conversation_id in replied_conv_ids OR flags & 0x4) / count
         - flagged_count: sum(flagged)
         - is_bidirectional: sender address exists in sent_recipients
         - times_sent_to / last_sent_to: from sent_recipients lookup
       - Return dict keyed by lowercase address

    2. `is_protected(message: Message, profile: ContactProfile, replied_conv_ids: set[int]) -> bool`:
       Protection per CONTEXT.md locked decision:
       - profile.is_bidirectional (sender is in Sent recipients) -> True
       - message.conversation_id in replied_conv_ids -> True
       - message.flags & 0x4 (replied bit) -> True
       - message.flags & 0x10 (forwarded bit) -> True
       - Otherwise False

    3. `check_protection_override(profile: ContactProfile) -> bool`:
       Per CONTEXT.md locked decision — ratio-based override:
       - If profile.read_rate < 0.05 (below 5%) -> True (override protection)
       - Otherwise False (protection holds)
       The override catches newsletters accidentally replied to once but never engaged with.

    4. `extract_behavioral_signals(message: Message, profile: ContactProfile) -> list[SignalResult]`:
       Extract per-message behavioral signals as SignalResult objects:
       - read_signal: 1.0 if read, 0.0 if unread (weight 0.15)
       - reply_signal: profile.reply_rate (weight 0.10)
       - flagged_signal: 1.0 if profile.flagged_count > 0, else 0.0 (weight 0.05)
       - automation_signal: 0.0 if automated_conversation > 0, 1.0 if human (weight 0.05). Note: inverted — higher value means more keep-worthy.
       - unsubscribe_signal: 0.0 if unsubscribe_type is not None, 1.0 if None (weight included in automation)
       Each signal includes an explanation string like "read=1 (message was read by user)"
