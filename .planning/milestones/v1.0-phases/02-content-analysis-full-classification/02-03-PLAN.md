  <name>Task 1: Fused classification engine with reclassification rules</name>
  <files>src/icloud_cleanup/classifier.py, tests/test_classifier.py</files>
  <behavior>
    - Test fuse_classification: blends metadata_confidence 0.6 and content_score 0.4 weights correctly
    - Test fuse_classification: returns single float in [0, 1]
    - Test reclassify_with_content: Review + high content_score (0.8) -> Keep tier
    - Test reclassify_with_content: Review + low content_score (0.2) -> Trash tier
    - Test reclassify_with_content: Review + mid content_score (0.5) -> stays Review
    - Test reclassify_with_content: Trash + high content_score (0.8) -> promoted to Review or Keep (safety net)
    - Test reclassify_with_content: Keep_Active + low content_score (0.1) -> stays Keep_Active (NEVER demoted)
    - Test reclassify_with_content: Keep_Historical + low content_score (0.1) -> stays Keep_Historical (NEVER demoted)
    - Test reclassify_with_content: protected message stays protected regardless of content score
    - Test reclassify_with_content: updates Classification fields (content_score, cluster_id, cluster_label, content_source)
  </behavior>
  <action>
    Add two new functions to `src/icloud_cleanup/classifier.py`:

    1. `fuse_classification(metadata_confidence: float, content_score: float, metadata_weight: float = 0.6, content_weight: float = 0.4) -> float`:
       - Simple weighted blend: `metadata_confidence * metadata_weight + content_score * content_weight`
       - Clamp to [0.0, 1.0]
       - Returns single fused confidence score

    2. `reclassify_with_content(classification: Classification, content_score: float, cluster_id: int, cluster_label: str, content_source: str, profile: ContactProfile, message: Message, replied_conv_ids: set[int]) -> Classification`:
       - Compute fused confidence via `fuse_classification(classification.confidence, content_score)`
       - Apply reclassification rules from CONTEXT.md decisions:
         a. If original tier is KEEP_ACTIVE or KEEP_HISTORICAL: DO NOT change tier (Keep->Trash is NEVER allowed). Update content fields only.
         b. If original tier is TRASH: allow promotion to REVIEW or KEEP if fused confidence is high enough (safety net against metadata false positives). Use same threshold logic as assign_tier but skip the trash path if content strongly disagrees.
         c. If original tier is REVIEW: full flexibility -- run assign_tier with fused confidence to get new tier.
       - Preserve protection status from original classification
       - Return new Classification with updated fields:
         - confidence = fused confidence
         - tier = reclassified tier (or original if locked)
         - signals = original signals + "; content_score={content_score:.2f}, cluster={cluster_label}"
         - content_score, cluster_id, cluster_label, content_source all populated
         - timestamp = current time

    Add test class `TestFusedClassification` and `TestReclassRules` to `tests/test_classifier.py` covering all reclassification rule combinations from behavior list above.
  </action>
  <verify>
    <automated>uv run pytest tests/test_classifier.py -x -v</automated>
  </verify>
  <done>fuse_classification blends scores correctly. reclassify_with_content enforces all tier transition rules: Keep never demoted, Trash can be promoted, Review has full flexibility. All content fields populated on output Classification.</done>
