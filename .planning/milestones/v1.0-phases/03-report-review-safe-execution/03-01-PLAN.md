Build the report generation and auto-triage modules that transform Phase 2 checkpoint data into structured reports and pre-filter the Review tier before human interaction.

Purpose: Report generation (EXEC-01) produces the three-format cleanup report users need to understand their mailbox state. Output: `report.py` with tier-first + cluster-detail report in terminal/JSON/Markdown formats; `auto_triage.py` with automated resolution passes and transparency summary. Both modules are pure data transformation with no UI interaction, enabling parallel development with Plan 02.

<task type="auto" tdd="true">
  <name>Task 2: Auto-triage pre-review resolution engine</name>
  <files>src/icloud_cleanup/auto_triage.py, tests/test_auto_triage.py</files>
  <behavior>
    - auto_triage identifies clusters where ALL emails have same tier and confidence > 0.85 (cluster unanimity)
    - auto_triage identifies senders where ALL emails have same tier and confidence > 0.80 (sender consistency)
    - Auto-resolved items get action="auto_resolved" with reason string
    - Remaining unresolved items returned separately for human review
    - AutoTriageResult contains auto_resolved count/clusters, remaining count/clusters, and per-resolution details for transparency
    - Protected emails are never auto-resolved to trash (safety check)
    - Cluster unanimity check handles noise cluster (cluster_id=-1 or None) by skipping it
  </behavior>
  <action>
    Create `src/icloud_cleanup/auto_triage.py`:

    1. `AutoTriageResult` dataclass:
       - `auto_resolved: list[AutoResolution]` (each has message_ids, action, reason, tier, cluster_label)
       - `remaining: list[Classification]` (items needing human review)
       - `auto_resolved_count: int`
       - `auto_resolved_cluster_count: int`
       - `remaining_count: int`
       - `remaining_cluster_count: int`

    2. `AutoResolution` dataclass:
       - `message_ids: list[int]`
       - `tier: Tier`
       - `reason: str` (e.g., "Cluster unanimity: all 150 emails classified as trash with avg confidence 0.92")
       - `cluster_label: str | None`
       - `avg_confidence: float`

    3. `auto_triage(classifications, review_only=True)` -> AutoTriageResult:
       - If review_only=True, only process Review-tier items (default behavior)
       - Pass 1 - Cluster unanimity: group by cluster_id (skip None/-1 noise), check if all items share same tier with confidence > 0.85. If yes, auto-resolve entire cluster
       - Pass 2 - Sender consistency: for remaining unresolved, group by sender (need message_id -> sender mapping, accept as parameter). Check if all items from sender share same tier with confidence > 0.80. If yes, auto-resolve that sender's group
       - Safety: if auto-resolved tier is TRASH, verify no protected=True items in the group. If any protected, skip that group
       - Return AutoTriageResult with resolved groups and remaining items

    Accept a `sender_lookup: dict[int, str]` parameter mapping message_id to sender_address (caller builds this from Message list to avoid coupling auto_triage to scanner).
  </action>

    - Protected emails never auto-resolved to trash
    - Noise cluster (None/-1) skipped in cluster unanimity pass
    - AutoTriageResult provides accurate counts and transparency details
    - Remaining items correctly excludes all auto-resolved message_ids
