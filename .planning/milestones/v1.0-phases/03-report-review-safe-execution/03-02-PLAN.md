<objective>
Build the safe execution engine (AppleScript + action log) and Claude API fallback module for ambiguous email classification.

Purpose: The executor (EXEC-03) is the critical safety layer -- it moves approved emails to Trash via AppleScript with full audit logging and dry-run default. The API fallback (EXEC-04) provides a last-resort classification path for emails that remain ambiguous after human review, using structured metadata summaries only (never raw bodies) with upfront cost transparency.

Output: `executor.py` with AppleScript generation, batch execution, SQLite action log, dry-run mode, and restore capability; `api_fallback.py` with metadata payload builder, cost estimator, batch API submission, and result integration back into checkpoint.

  <name>Task 1: Executor module -- AppleScript generation, action log, dry-run, batch execution</name>
  <files>src/icloud_cleanup/executor.py, tests/test_executor.py</files>
  <behavior>
    - generate_applescript produces correct AppleScript syntax for moving message by ROWID from source mailbox to "Deleted Messages"
    - generate_applescript correctly converts mailbox_url to AppleScript mailbox reference (URL-decode, strip imap://UUID/ prefix)
    - ActionLog creates SQLite DB with action_log table and indexes on first use
    - ActionLog.log_action records message_id, rowid, subject, sender, tier, confidence, action, source_mailbox, timestamp, dry_run flag, success, error_message
    - ActionLog.get_actions queries actions by filters (action type, dry_run, timestamp range)
    - Dry-run mode logs all actions with dry_run=True and does NOT call osascript
    - execute_deletions processes messages in batches of --batch-size (default 100) with configurable pause between batches
    - execute_deletions returns summary: success_count, error_count, errors list
    - restore_from_log generates AppleScript to move messages back from "Deleted Messages" to original mailbox
    - Protected messages are rejected from execution with error (safety net)
  </behavior>
  <action>
    Create `src/icloud_cleanup/executor.py`:

    1. `url_to_applescript_mailbox(mailbox_url: str) -> str`:
       - Strip `imap://{uuid}/` prefix using split or regex
       - URL-decode the remaining path with `urllib.parse.unquote`
       - Return `mailbox "{path}" of account "iCloud"`
       - Handle edge cases: empty path, nested folders

    2. `generate_applescript(rowid: int, source_mailbox: str, trash_mailbox: str = "Deleted Messages") -> str`:
       - Build AppleScript that finds message by `id` in source mailbox and sets its mailbox to trash
       - Source mailbox is already an AppleScript reference string (output of url_to_applescript_mailbox)
       - Use the verified pattern: `set mailbox of item 1 of matchedMsgs to trashMailbox`

    3. `generate_restore_script(rowid: int, original_mailbox: str) -> str`:
       - Reverse operation: find message in "Deleted Messages", move back to original mailbox

    4. `ActionLog` class:
       - `__init__(self, db_path: Path)` -- creates SQLite DB and action_log table if not exists (use schema from 03-RESEARCH.md)
       - `log_action(self, ...)` -- insert row
       - `get_actions(self, action: str | None, dry_run: bool | None, limit: int = 100) -> list[dict]` -- query with optional filters
       - `get_restorable(self) -> list[dict]` -- actions where action='move_to_trash', success=True, dry_run=False
       - Uses context manager for connection handling

    5. `execute_deletions(messages, classifications, action_log, dry_run=True, batch_size=100, batch_pause=2.0) -> dict`:
       - Takes list of (Message, Classification) tuples for approved deletions
       - Safety check: reject any message where classification.protected is True (log error, skip)
       - For each message: convert mailbox_url -> AppleScript ref, generate script
       - If dry_run: log with dry_run=True, do NOT call subprocess
       - If not dry_run: call `subprocess.run(["osascript", "-e", script], capture_output=True, text=True, timeout=30)`
       - Log result (success/failure/error_message)
       - After each batch_size messages, `time.sleep(batch_pause)` to let Mail.app sync
       - Return {"success_count": N, "error_count": N, "skipped_protected": N, "errors": [...]}

    6. `restore_from_log(action_log, dry_run=True, batch_size=100, batch_pause=2.0) -> dict`:
       - Get restorable actions from log
       - Generate restore scripts, execute (or dry-run)
       - Log restore actions

    Tests should mock subprocess.run to avoid actual AppleScript execution. Test the SQL schema creation, log queries, AppleScript string generation, URL conversion, batch chunking, and protected-message rejection.

    - Dry-run mode logs actions without calling osascript
